# Issue Tracker: Local Markdown

Issues in this repository are tracked as markdown files within the `.scratch/` directory.

## Workflow

1. **Creation**: Issues are created as `.md` files in `.scratch/<feature-name>/<issue-id>-<title>.md`.
2. **Reading**: Skills read these files to understand the current backlog and task status.
3. **Updating**: Status changes and metadata updates are performed by editing the file headers.
