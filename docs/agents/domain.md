# Domain Documentation

This project uses a single-context layout for domain and architectural documentation.

## Locations

- **Domain Language**: `CONTEXT.md` at the root contains the project's glossary and domain-specific concepts.
- **Architectural Decisions**: `docs/adr/` contains Architectural Decision Records (ADRs) tracking significant technical choices.

## Consumer Rules

- Always read `CONTEXT.md` before starting work to ensure correct terminology.
- Check `docs/adr/` for constraints or patterns that must be followed in the current area of work.
